## 🎯 What We're Looking For

We welcome high-quality contributions related to Vibecoding and AI-assisted programming:

### ✅ What We Accept

- **Learning Tutorials** - Step-by-step guides and hands-on projects
- **Learning Paths** - Structured curriculum for different skill levels
- **Resource Recommendations** - Books, courses, articles, tools
- **Documentation Improvements** - Fixes, clarifications, translations
- **Issue Reports** - Bugs, broken links, outdated content

### 📋 Submission Types

#### 📖 Guides & Tutorials

- Original tutorials with clear structure
- Step-by-step instructions with code examples
- Suitable for specific skill levels (beginner/intermediate/advanced)
- Include expected outcomes and prerequisites

#### 🗺️ Learning Paths

- Structured curriculum design
- Progressive difficulty levels
- Clear milestones and objectives
- Recommended time estimates

#### 📚 Resource Recommendations

- Quality learning materials (free or paid)
- Clear description of what the resource covers
- Target audience level
- Why it's valuable for learners

## 🚀 How to Contribute

### Step 1: Choose the Contribution Type

| Type | Description | How to Submit |
|------|-------------|---------------|
| New Tutorial | Add a complete tutorial | Create PR or Issue |
| Learning Path | Propose a learning roadmap | Issue: Learning Path template |
| Resource | Recommend learning material | Issue: Resource Request template |
| Documentation | Fix docs or add translations | Pull Request |
| Bug Report | Report issues | Issue: Resource Bug template |

### Step 2: Prepare Your Contribution

For **Tutorials**:
- Title and description
- Target audience level
- Prerequisites
- Step-by-step content with code examples
- Expected outcomes

For **Learning Paths**:
- Topic/theme
- Target audience
- Learning objectives
- Chapter outline with estimated time
- Reference materials

For **Resources**:
- Resource name and type (book/course/article/tool)
- Link to the resource
- Brief description
- Target level
- Why it's recommended

### Step 3: Create an Issue or Pull Request

1. **For new content**: Use the appropriate [Issue Template](.github/ISSUE_TEMPLATE/)
2. **For documentation fixes**: Create a Pull Request directly
3. **For discussions**: Use [GitHub Discussions](https://github.com/trae-community/trae-learning-projects/discussions)

### Step 4: Review Process

- **Acknowledgment**: Within 24 hours
- **Review**: 3-7 business days
- **Feedback**: If changes needed, we'll comment on your issue/PR
- **Merge**: Once approved, your contribution will be added!

## 📝 Content Standards

### Quality Requirements

#### Documentation Quality
- Clear and concise language
- Proper formatting (Markdown)
- Helpful code examples
- No broken links

#### Content Quality
- Accurate and up-to-date information
- Original content (or properly attributed)
- Appropriate for the target audience
- Well-structured with clear headings

### Formatting Guidelines

#### Tutorial Structure
```markdown
# Tutorial Title

## Prerequisites
- What readers need to know beforehand

## Objectives
- What readers will learn

## Step 1: [Title]
[Content with code examples]

## Step 2: [Title]
[Content with code examples]

## Conclusion
- Summary of what was covered
- Next steps for further learning
```

#### Learning Path Structure
```markdown
# Learning Path: [Topic]

## Overview
[Brief description of the learning path]

## Target Audience
[Who this is for]

## Estimated Time
[Time to complete]

## Prerequisites
[What readers should know before starting]

## Modules
### Module 1: [Title]
- Topics covered
- Estimated time

### Module 2: [Title]
- Topics covered
- Estimated time

## Resources
[Recommended materials for each module]
```

## 💡 Tips for Successful Contributions

### ✅ Do's

- ✅ Search existing issues/PRs first to avoid duplicates
- ✅ Provide complete information in your submission
- ✅ Use clear, descriptive titles
- ✅ Include code examples where relevant
- ✅ Test your content before submitting
- ✅ Respond to review feedback promptly

### ❌ Don'ts

- ❌ Submit incomplete tutorials or drafts
- ❌ Include broken or invalid links
- ❌ Copy content without attribution
- ❌ Submit promotional or spam content
- ❌ Duplicate existing resources without adding value

## 🤔 Common Questions

### Q: Can I submit multiple tutorials?
A: Yes! Each tutorial should be submitted as a separate PR or issue.

### Q: Can I translate existing content?
A: Absolutely! We welcome translations. Please check existing translations first.

### Q: Is there a format requirement?
A: We recommend following our content templates. Check the examples above for guidance.

### Q: Who owns the content I submit?
A: You retain all rights to your original content. By submitting, you agree to license it under the project's MIT license.

### Q: How long does review take?
A: We aim to respond within 24 hours and complete reviews within 7 business days.
