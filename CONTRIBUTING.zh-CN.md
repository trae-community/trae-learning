## 🎯 我们需要什么样的贡献

我们欢迎与 Vibecoding 和 AI 辅助编程相关的高质量贡献：

### ✅ 接受的贡献类型

- **学习教程** - 循序渐进指南和动手实践项目
- **学习路线** - 针对不同技能水平的系统课程
- **资源推荐** - 书籍、课程、文章、工具推荐
- **文档改进** - 修复、澄清、翻译
- **问题报告** - Bug、失效链接、过时内容

### 📋 提交类型说明

#### 📖 指南与教程

- 结构清晰的原创教程
- 包含代码示例的逐步说明
- 适合特定技能水平（入门/进阶/高级）
- 包含预期成果和前置要求

#### 🗺️ 学习路线

- 系统化的课程设计
- 渐进式难度级别
- 明确的里程碑和目标
- 建议的学习时间估算

#### 📚 资源推荐

- 优质学习材料（免费或付费）
- 清晰的内容描述
- 目标受众级别
- 对学习者的价值说明

## 🚀 如何贡献

### 步骤 1: 选择贡献类型

| 类型 | 说明 | 提交方式 |
|------|------|----------|
| 新教程 | 添加完整教程 | PR 或 Issue |
| 学习路线 | 提出学习路线图 | Issue: Learning Path 模板 |
| 资源推荐 | 推荐学习材料 | Issue: Resource Request 模板 |
| 文档修复 | 修复文档或添加翻译 | Pull Request |
| 问题报告 | 报告问题 | Issue: Resource Bug 模板 |

### 步骤 2: 准备您的贡献

**对于教程**：
- 标题和描述
- 目标受众级别
- 前置要求
- 包含代码示例的逐步内容
- 预期成果

**对于学习路线**：
- 主题/方向
- 目标受众
- 学习目标
- 章节大纲和时间估算
- 参考资料

**对于资源推荐**：
- 资源名称和类型（书籍/课程/文章/工具）
- 资源链接
- 简要描述
- 目标级别
- 推荐理由

### 步骤 3: 创建 Issue 或 Pull Request

1. **新内容**：使用相应的 [Issue 模板](.github/ISSUE_TEMPLATE/)
2. **文档修复**：直接创建 Pull Request
3. **讨论**：使用 [GitHub Discussions](https://github.com/trae-community/trae-learning-projects/discussions)

### 步骤 4: 审核流程

- **确认**：24 小时内
- **审核**：3-7 个工作日
- **反馈**：如需修改，我们会在您的 issue/PR 上留言
- **合并**：审核通过后，您的贡献将被添加！

## 📝 内容标准

### 质量要求

#### 文档质量
- 清晰简洁的语言
- 正确的格式（Markdown）
- 有帮助的代码示例
- 无失效链接

#### 内容质量
- 准确且最新的信息
- 原创内容（或正确注明来源）
- 适合目标受众
- 结构清晰、标题明确

### 格式指南

#### 教程结构
```markdown
# 教程标题

## 前置要求
- 读者需要预先了解的内容

## 学习目标
- 读者将学到什么

## 步骤 1: [标题]
[包含代码示例的内容]

## 步骤 2: [标题]
[包含代码示例的内容]

## 总结
- 本教程涵盖的内容总结
- 进一步学习的下一步
```

#### 学习路线结构
```markdown
# 学习路线：[主题]

## 概述
[学习路线的简要描述]

## 目标受众
[适合谁学习]

## 预计时间
[完成所需时间]

## 前置要求
[开始学习前应该掌握的知识]

## 模块
### 模块 1: [标题]
- 涵盖的主题
- 预计时间

### 模块 2: [标题]
- 涵盖的主题
- 预计时间

## 资源
[每个模块推荐的学习材料]
```

## 💡 成功贡献的技巧

### ✅ 推荐做法

- ✅ 先搜索现有的 issue/PR，避免重复
- ✅ 提交时提供完整信息
- ✅ 使用清晰描述性的标题
- ✅ 相关时包含代码示例
- ✅ 提交前测试内容
- ✅ 及时回复审核反馈

### ❌ 避免做法

- ❌ 提交不完整的教程或草稿
- ❌ 包含失效或无效链接
- ❌ 未注明来源地复制内容
- ❌ 提交推广或垃圾内容
- ❌ 在不增加价值的情况下重复现有资源

## 🤔 常见问题

### 问：我可以提交多个教程吗？
答：可以！每个教程应作为单独的 PR 或 issue 提交。

### 问：我可以翻译现有内容吗？
答：当然可以！我们欢迎翻译。请先检查现有的翻译。

### 问：有格式要求吗？
答：我们建议遵循我们的内容模板。请查看上面的示例作为指导。

### 问：我提交的内容归谁所有？
答：您保留对原创内容的全部权利。提交即表示您同意将其归入项目的 MIT 许可证。

### 问：审核需要多长时间？
答：我们会在 24 小时内回复，并在 7 个工作日内完成审核。
