# 心流与效率

写代码时最消耗状态的，往往不是"想不出来"，而是那些被迫打断的瞬间：忘了某个 API 的参数顺序、查了半小时文档发现走错了路、手打出一堆样板代码才能开始真正的逻辑……Vibecoding 的核心价值之一，就是消除这些摩擦。

## 什么在打断你的心流

- **语法和 API 查询**：不是不会，但要停下来查
- **样板代码**：每次新开一个文件都要先写一堆才能到正题
- **不确定性焦虑**：这个方案对不对？有没有更好的做法？

这三类问题加在一起，能把一个 2 小时的任务拖成一整天。

## Vibecoding 如何改变这件事

AI 处理的不是"替你想"，而是帮你消除这些中断点：

1. **Prompt 描述意图，AI 写样板** — 你说"我要一个带分页的用户列表 API"，框架搭好了，你只需要填业务逻辑
2. **不确定时直接问** — 不用切换标签页，在 Trae 里直接问"这两种方案哪个更适合我的场景"
3. **边写边迭代** — 不满意直接说"把错误处理加全"，不需要从头改

## 保持高效的几个实际习惯

**拆小任务**：一次只做一件事。"写完整个后端"会让人不知从哪下手；"先实现 POST /users" 清晰得多。

**Prompt 先于代码**：动手前花 30 秒把需求整理成一句话，给 AI 也给自己。这个过程往往会帮你发现还没想清楚的地方。

**不要盲接**：AI 生成的代码要看一眼再用。不是因为不信任，而是审查本身会让你真正理解这段代码在做什么。

**及时提交**：每完成一个小功能就 commit。有退路的时候，你会更敢于实验。
